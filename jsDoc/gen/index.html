#UI

role

  • 기획, 디자인에서 발생하는 기본 html UI를 사용할 수 없는 경우 커스텀 UI를 제공한다.
  • 개발이 없는 정적인 페이지는 퍼블리싱, ui스크립터가 작업한다.
  • 개발과 섞인 동적인 ui는 ui스크립터가 작업하고 사용방법을 제공해준다.
  • 공통ui는 별도의 페이지에서 예제를 제공하고 각 페이지에 독립적으로 들어가는 ui는 페이지내에서 예제를 제공한다. (각 예제는 단순 예제일뿐 사용의도에 맞게 id변경등이 필요할수있다.)
  • 개발친화적인 ui는 제공하지 않는다.(금액에 콤마 찍어주는 등 벨리데이션과 관련된 요소는 제외)
  • 벨리데이션은 개발 프로세스의 이해가 필요하므로 ui스크립터가 작업하지 않는다.
  • 모든 스크립트는 ui구조가 정해져 있으므로 임의로 요소를 수정하지 않고 퍼블리셔 pl 또는 ui스크립터와 상의 한다.

정의

  • 모든 스크립트 폼요소를 제외한 ui는 ui-xxx 클래스를 갖는다. (예: ui-accordion, ui-tab, ui-popup 등등)
  • 모든 ui는 페이지 로드시 기본 호출이 되어있고
  • 해당 ui를 재호출 할수있다.
  • 아래와 같이 호출하면 해당 ui의 정보를 확인할수 있고 제공된 메소드를 호출할수 있다. (메소드는 제공된것만 사용하고 이외의 것을 사용할일이 있으면 ui스크립터와 상의한다.)
<!-- html -->
<div class="popup-group">
    <div class="dimm"></div>
    <div class="popup-dialog ui-popup ui-full" id="full-popup-01">
        <!-- 팝업내용 -->
    </div>
</div>

// js
$('#full-popup-01').data('ui')
  • 모든 ui는 옵션을 가질 수 있으며 옵션은 해당 요소의 data-xx로 값을 넣는다.
  • 아래 예제의 경우 tab의 기본 보이는 index가 1일 경우 아래처럼 처리함을 보여준다.
<!-- html -->
<div class="tab-nav-container ui-tab" data-no="1">
    <!-- 탭내용 -->
</div>

요소의 변경과 값 입력

ui스크립트는 기본 jquery를 사용하여 제작되었고 vue, react처럼 돔전체를 관리하진 않으므로 페이지 로드시점 이후 동적으로 요소를 조작하거나 값을 입력시 .update()를 호출하여 요소를 업데이트 해주거나 값을 입력하고 화면에 반영해주려면 업데이트를 해줘야한다 (jquery val(), prop(), attr()값을 넣어줄경우는 제외 jquery hook으로 자동 업데이트됨)

  // ui스크립트 외부에서 돔조작
  $('#some-ui').data('ui').update();    // 필수 호출

UI의 초기화

  • 기본적으로 페이지가 로드될때 해당요소가 있으면 자동으로 초기화 된다.
  • 하지만 동적으로 컨텐츠가 들어오고 나서 ui의 추가적인 초기화가 필요할때 아래와 같이 처리한다.
  hkUI.refreshUI()  // 새로들어온 요소중에 초기화할 ui 초기화
  hkUI.refreshUI(true)  // [비추천]기존에 로드된 ui까지 모두 초기화

팝업

정의

종류

  • [정적]풀 팝업(.ui-full)
    • 높이가 100%인 컨텐츠를 모두 덮는 팝업
  • [정적]심플 팝업(.ui-simple)
    • 높이가 100%가 아닌 딤드와 컨텐츠가 보이는 팝업
  • [정적]보톰 팝업(.ui-bottom)
    • 하단에서 올라오는 팝업
  • [동적]셀렉트 보톰팝업(.ui-bottom)
    • 보톰 팝업과 룩은 같지만 기존 select요소 기능을 하는 팝업
  • [동적]동영상 팝업(.ui-video)
    • 동영상 단독으로 보이는 팝업
  • [동적]날짜 팝업(.ui-bottom)
    • 보톰 팝업과 같으나 내용이 datepicker
  • [동적]앨럿 팝업
    • 네이티브 alert, confirm을 대채하는 팝업

동적생성과 정적생성

  • 각페이지에서 생성이 번거로운 셀렉트 보톰팝업과 앨럿팝업은 동정생성되므로 사용법에 맞게 사용하면되고 풀팝업, 심플팝업, 보톰팝업은 기본적으로 퍼블리싱되어 해당요소에 위치하게 되어있고 스크립트는 열고 닫고만 제공한다. 각 팝업에 맞는 개발을 한 후 열고 닫으면 된다.

팝업 종류에 맞게 클래스 사용

  • 모든 팝업은 ui-popup클래스가 붙으며 종류에 따른 클래스가 붙는다.
  • 풀팝업: ui-full, 심플팝업: ui-simple, 보톰팝업: ui-bottom, 앨럿팝업: ui-alert

팝업 요소의 위치

  • html > body > #wrap > .popup-group
  • 팝업은 모두 해당 요소에 위치해야하고 다른곳에 있을 수 없다.

dimm의 사용

  • 딤드는 따로 개발에서 호출하지 않고 ui스크립트에서 관리한다.
  • 중첩 팝업일 경우 dimm한개로 관리해야하므로 ui스크립트에서 관리

예제

hkUI.alert.open({
    header: '제목',   // 선택
    content: '컨텐츠', // 필수
    btnCancel: {      // 선택
        text: 'cancel', // 선택(기본값: 취소)
        fn: function(){
            // callback
            console.log(1);
        }
    },
    btnConfirm: {     // 선택
        text: 'ok', // 선택(기본값: 확인)
        fn: function(){
            // callback
            console.log(2);
        }
    }
});

// ex) 컨텐츠와 확인버튼만 있고 버튼명이 확인일경우
hkUI.alert.open({

    }
  }
});

옵션

종류
  • data-no
    • 0을 포함한 자연수 : initialSlide 초기값 설정
  • data-nav
    • true : 화살표 네비게이션 버튼 사용
    • false : 사용안함
  • data-paging
    • true : 페이지네이션 사용
    • false : 사용안함
  • data-paging-type
    • fraction : 숫자 타입 페이지네이션 사용
    • undefined : 사용안함
  • data-paging-title
    • 텍스트 : 페이지네이션의 제목
    • undefined : 사용안함
  • data-slides-per-view
    • auto : 한번에 보여줄 슬라이드 갯수 자동 설정
    • 자연수 : 한번에 보여줄 슬라이드 갯수
  • data-space-between
    • 0을 포함한 자연수 : 슬라이드 사이 여백 값 (px 기준)
디폴트 옵션
  • data-no="0" data-nav="false" data-paging="false" data-paging-type="undefined" data-paging-title="undefined" data-slides-per-view="auto" data-space-between="20"
